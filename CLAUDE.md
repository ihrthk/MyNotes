# CLAUDE.md







This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

这是一个使用 [Obsidian](https://obsidian.md/) 管理的个人知识库，采用 Git 进行版本控制。笔记内容包括个人生活、工作学习、待办事项、财务管理、人脉资源等分类。

## 目录结构

```
MyNotes/
├── 01-个人生活/          # 个人生活相关笔记
├── 02-工作学习/          # 工作和技术学习笔记
├── 03-待办事项/          # 待办事项和任务列表
├── 04-财务管理/          # 财务管理、投资理财、保险
├── 05-人脉资源/          # 人脉联系人、阅读列表
├── 06-数据存储/          # 数据库文件存储
├── 07-快捷笔记/          # 快速笔记
├── archives/             # 归档文件
└── assets/               # 全局资源目录
```

## 文件组织规则

### Markdown 文件命名

- **扁平化规则**：如果一个目录只包含一个 Markdown 文件，应该将文件移到父目录，删除子目录
- **命名格式**：`目录名 文件名.md`（例如：`南京工作生活 笔记.md`）
- **哈希后缀**：从 Notion 导出的文件带有哈希后缀（如 `abc123.md`），这些是历史遗留，新文件不应使用此格式

### 附件资源管理

每个目录级别只有一个 `assets/` 目录，通过子文件夹区分不同笔记的附件：

```
01-个人生活/
├── 南京工作生活.md
├── 炮轰钉钉.md
└── assets/
    ├── 南京工作生活/
    │   ├── image.png
    │   └── document.pdf
    └── 炮轰钉钉/
        └── screenshot.jpg
```

**引用路径格式**：
```markdown
![图片描述](assets/南京工作生活/image.png)
[文档标题](assets/南京工作生活/document.pdf)
```

**重要**：
- 同一目录下的所有 MD 文件共享一个 `assets/` 目录
- assets 内按 MD 文件名建立子文件夹分类
- 引用路径使用相对路径 `assets/文件名/附件名`
- **中文路径处理**：Obsidian 可能对中文路径进行 URL 编码（如 `%E5%8D%97%E4%BA%AC`），在手动编辑链接时需要注意处理这种编码格式

## Obsidian 配置

### 基础配置
- **链接格式**：使用 Markdown 链接（`useMarkdownLinks: true`）
- **新链接格式**：相对路径（`newLinkFormat: "relative"`）

### 已启用的核心插件
- **每日笔记**：按日期创建日记
- **模板**：笔记模板系统
- **属性**：文件元数据管理
- **大纲**：文档大纲视图
- **书签**：快速访问常用文件
- **画布**：可视化笔记关联
- **Git 同步**：与 Git 仓库集成

### 社区插件
- **obsidian-git**：自动将笔记备份到 Git 仓库
- **obsidian-custom-attachment-location**：自定义附件存储位置

## 文件操作注意事项

### 移动或重命名文件时

1. **同时移动对应的 `.assets` 目录**
2. **更新 Markdown 文件中的引用路径**
3. **注意 URL 编码**：Obsidian 可能对中文路径进行 URL 编码（如 `%E5%8D%97%E4%BA%AC`），更新路径时需要处理

### 删除文件时

1. **同时删除对应的 `.assets` 目录**
2. **检查其他文件是否有对该文件的引用**

### 添加新附件时

1. 将附件放入对应 MD 文件的 `assets/笔记名/` 目录
2. 使用相对路径引用

## 常见工作流程

### 创建新笔记
1. 在相应分类目录下创建 Markdown 文件
2. 如有附件，在同级 `assets/` 目录下创建同名子文件夹
3. 使用相对路径引用附件资源
4. Git 会自动备份，或手动提交更改

### 移动/重命名笔记
1. 同时移动对应的 `assets/笔记名/` 目录
2. 更新 Markdown 文件中的资源引用路径
3. 检查其他文件是否有对该文件的引用并更新
4. 注意处理中文路径的 URL 编码问题

### 删除笔记
1. 同时删除对应的 `assets/笔记名/` 目录
2. 检查其他文件是否有对该文件的引用
3. 清理 `.trash/` 目录中的废弃文件

## Git 工作流程

### 自动备份机制
obsidian-git 插件会自动创建备份提交，提交历史中可以看到：
```
vault backup: 2026-04-20 10:39:21
vault backup: 2026-04-20 10:38:17
```

### 手动操作建议
1. **操作前检查**：在进行手动 Git 操作前，先检查并拉取自动备份的提交
2. **提交信息规范**：手动提交使用描述性的中文提交信息
3. **避免冲突**：避免在自动备份期间进行手动操作

### Git 提交规范
- 使用中文提交信息
- 提交前检查是否有未跟踪的 assets 文件
- 大型附件（PDF、图片等）应该被提交
- 注意处理自动备份与手动提交的冲突

- `.obsidian/`：Obsidian 配置目录，包含插件和界面设置（不要手动修改）

- `.claude/`：Claude Code 配置目录
- `assets/`：全局资源目录，存储根目录笔记的附件
